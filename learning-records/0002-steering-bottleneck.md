# Core bottleneck: steering AI tools and verifying output

The user identified their main blocker as: "cannot steer AI in right direction, debug things." This breaks into three sub-problems: (1) not knowing what to ask for precisely enough, (2) not being able to verify if the output is correct, (3) not knowing how to describe what's wrong to get a fix.

**Implications**: Every lesson should include a "predict before you run" exercise — force an expected output before running code. Teach the pattern: state intent → predict output → run → if wrong, describe expected vs. actual (never just "fix it").
