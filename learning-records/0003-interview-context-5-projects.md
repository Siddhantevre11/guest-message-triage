# Preparing for Besty technical interview — 5 projects, ~45 min each

The user is building toward a technical interview at Besty (AI-powered guest communication SaaS). There are 5 projects to demonstrate: triage/routing, upsell detection with tool use, policy RAG with self-correction, memory + failure recovery, and pricing planner. Each has a 30-minute requirement change mid-build.

**Implications**: Lessons should sequence by project order (Project 1 first). The user needs to be able to articulate tradeoffs out loud — why agent vs. deterministic, what breaks at scale — so lessons should include "say this out loud" prompts, not just code.
