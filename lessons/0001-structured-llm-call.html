Your First Structured LLM Call

Lesson 01 · Project 1 — Guest Message Triage

Every project you're building — triage, upsell detection, policy Q&A — is the same primitive under the hood: give an LLM a message, get back structured data. Master this once and the rest is wiring.

The Core Concept

When you call an LLM normally, it returns free text. That's fine for conversations, but useless when you need to route a message to a handler. You need a guaranteed shape every time.

model.with_structured_output(Schema) wraps the LLM so it can only return data that matches your schema. If the model tries to return something that doesn't fit, the API rejects and retries internally — not your problem.

The guarantee: Prompting for JSON gives you free text that looks like JSON. Structured output gives you a Python object with real types, every call. This is why ADR-0001 chose it.

The Pattern

Three lines is all there is:

llm = ChatGroq(model="llama3-8b-8192", temperature=0)
classifier = llm.with_structured_output(ClassificationOutput)
result = classifier.invoke(messages)

result is now a ClassificationOutput object — not a string, not a dict. Real fields you can read with result.category.

The Full Code

import os
from langchain_groq import ChatGroq
from pydantic import BaseModel, Field
from typing import Literal

# 1. Define the output schema
class ClassificationOutput(BaseModel):
    category: Literal["maintenance", "booking", "complaint", "other"]
    confidence: float = Field(ge=0.0, le=1.0)
    summary: str = Field(description="One sentence: what does the guest want?")
    suggested_action: Literal[
        "dispatch_maintenance",
        "handle_booking",
        "handle_complaint",
        "handle_other",
        "escalate",
    ]

# 2. Create the structured LLM
llm = ChatGroq(model="llama3-8b-8192", temperature=0)
classifier = llm.with_structured_output(ClassificationOutput)

# 3. Classify a message
SYSTEM = """You classify guest messages for a short-term rental host.
Categories: maintenance (physical issue), booking (reservation logistics),
complaint (dissatisfaction), other (small talk, thanks, general questions).
Set confidence low (under 0.7) when the message is ambiguous."""

def classify(message: str) -> ClassificationOutput:
    return classifier.invoke([
        {"role": "system", "content": SYSTEM},
        {"role": "user", "content": message},
    ])

# 4. Try it
result = classify("The AC isn't working in my room.")
print(result.category)        # maintenance
print(result.confidence)      # 0.95 (or similar)
print(result.summary)         # "Guest reports AC is not working."
print(result.suggested_action) # dispatch_maintenance

Exercise — Predict Before You Run

The rule: Before running each line below, write down what you expect result.category to be. Then run it. If it doesn't match — describe what you expected vs. what you got. Don't just ask to fix it until you can say why it's wrong.

Add these five calls to your file, one at a time:

# 1. Clear maintenance
classify("The shower drain is completely blocked.")

# 2. Clear booking
classify("Can I check out an hour late tomorrow?")

# 3. Angry complaint
classify("This place is disgusting. I want a refund immediately.")

# 4. Low-stakes other
classify("Thanks so much for everything, we had a wonderful stay!")

# 5. Ambiguous — what does the model do?
classify("There's a strange smell coming from the vents — is that normal?")

For message 5: does it get maintenance or complaint? Check result.confidence — is it below 0.7?
